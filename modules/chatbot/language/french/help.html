ChatBot 1.0

Bienvenue dans Chatbot, le module de chat robotisé pour votre site Xoops.
Ce module est basé sur deux scripts en Javascripts qui simulent la conversation humaine, et donc une forme d'intelligence artificielle (IA).


Fonctionnalités proposées :
  • Nombre illimité de chatbot, disposant chacun d'expressions et d'un visuel personnalisées (couleurs, images, etc.) et visible en fonction des groupes d'appartenance des utilisateurs.
  • Nombre illimité de sujets de conversation, que l'on peut affecter au bot de son choix.
  • Nombre illimité de question/réponses avec mots clés. Script Akali.
  • Nombre illimité de question/réponses avec système de cache. Skript ElizaBot.
  • Système de rapport de discussion soumis par les utilisateurs pour adapter les questions réponses des bots.

Instructions :

  • Créez votre bot. Complétez les informations nécessaires, et ajoutez les phrase personnalisées.
    • Nom : nom du bot
    • Description :description du bot. S'affichera dans le bloc étendu, la page d'index et la page du bot.
    • Avatar : image représentant le bot. Stocker l'image dans le répertoire défini dessous.
    • Répertoire : contient toutes les données reltives au bot. Le répertoire ainsi sélectionné doit être ouvert en écriture afin de permettre la mise à jour des données relatives au bot.
    • Images de fond : nom des images employée pour l'affichage des bots. 2 images : image de la page d'index et image en fond d'écran de la boîte de dialogue. S'écrit de la façon suivante : index_bkg.png|textbox_bkg.png
    • Couleurs : couleurs employées sur la page du bot. S'écrit de la façon suivante : Texte de la page|Fond de page|Texte boîte de dialogue|Fond de boîte de dialogue|Epaisseur du cadre
    • Phrase d'accueil : chaque fois que le bot est activé, il affiche l'une des phrases choisie au hasard. Séparer chaque phrase par un |.
    • Phrase personnalisée : Indiquez ici quelques expressions types du bot. Ces expressions s'afficheront de façon alétoire lorsque le bot voudra changer de sujet de conversation.
    • Phrase d'incompréhension : Indiquez ici quelques réponses d'étonnement lorsque l'utilisateur se contente de valider des réponses vides.
    • Phrase de conclusion : Lorsque le bot arrive à bout d'argument sur un thème de discussion, il afficher l'une des phrases indiquée ici.
    • Groupes :Permission d'accès par groupe. Détermine quels utilisateur ont accès au bot.
    • Sujet de conversation : Sélectionnez ici les sujets de conversation que maîtrise le bot. Voir plus bas pour les 4 modes d'utilisation disponibles.
      Astuce : N'oubliez pas d'éditer votre bot et d'ajouter les thèmes de discussion que vous auriez créé ultérieurement !

  • Déposer les images dans le répertoire indiqué dans le profile du bot.

  • Créer les sujets de conversations. 4 modes possibles
    • Actif : sujet listé dans le profile du bot
    • Masqué : sujet utilisé par le bot, mais non listé dans son profile
    • Contextuel : utilisé avec le bloc. La conversation est activée en fonction de la navigation sur le site.
    • Inactif : le sujet est désactivé et ignoré par le bot.

  • Créer les question/réponse : Q/R Akali.
    Basé sur un système de calcul de points en fonction des occurences de mots clés, le bot renverra la réponse adéquate en fonction des questions posées.
    • Choisir un sujet de discussion auquel la question/réponse sera affectée.
    • Obligatoire : liste des mots clés obligatoires pour activer la réponse. Un seul terme correpondant activera la réponse.
    • Facultatif : liste des mots clés facultatifs pour induire une réponse. Un terme correspondant favorisera la réponse.
    • Contextuel : liste des mots clés pour lier des réponses entre-elles. Si aucun mot clé obligatoire n'est détecté dans la réponse, il poursuit la conversation sur un thème équivalent.
    • Réponses : Réponses fournies obligatoirement si l'un des mots clés est activé. A employer avec mots clés uniquement.
    • Discussion : Réponses fournies à une question précise, ou si aucun terme n'est détecté afin de relancer la discussion sur un autre sujet. Peut s'employer avec ou sans mots clés.

  • Créer les question/réponse : Q/R Eliza.
    Ces questions réponses sont automatiquement affectées à tous les bots. Le script se base sur la méthode 'Eliza', c'est à dire quelle renvoie à l'utilisateur des réponses basées ou incluant les mots clés repérées dans la question de l'utilisateur. Ce script fonctionne sur base d'un système de masque et peut être employé pour éluder les questions trop personnelles.
    • Questions : liste des masques à appliquer aux questions de l'utilisateur.
    • Réponses: liste des réponses potentielles, choisies de façon aléatoire. Utiliser $1 pour afficher les termes repris dans le masque de la question.

Les bonus : 

  •  Les émoticones
    Pour activer les émoticônes ou plus simplement afficher une image différente en fonction de la réponse du bot, nommez vos images au format .gif en fonction des mots clés pour lesquels ils doivent s'activer et déposez simplement les dans le répertoire de stockage du bot.
    Exemple : conduire.gif s'affichera lorsque le bot dire : "J'aime conduire."

  • Infos du site
    Dans les préférences du module, vous trouverez un champ "Info du site". Ce champ permet d'aller récupérer des informations dans la base de donnée du site en cours et d'en appeler le résultat en fonction de mots clés définis.
    La formule employée est la suivante : MOTS CLES,TABLE,COLUMN,WHERE. Veillez à séparer chaque terme par une virgule.
    • Exemple 1 : chatbots chatbot nom,chatbot_bot,bot_name,status=1 affichera le nom des bots actifs du module lorsque l'utilsateur entrera la question : "Quel est le nom des chatbots actifs de ce site ?"
    • Exemple 2 : chatbots Combien combien,chatbot_bot,count(*),status=1 affichera le nombre de bots actifs du module lorsque l'utilsateur entrera la question : "Combien de chatbots sont actifs sur ce site ?".

      Astuces : le premier mot clé est employé dans la réponse fournie. N'importe quelle table de la base de donnée du site peut ainsi être consultée, en ce compris les autres modules.
